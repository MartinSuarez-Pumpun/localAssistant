## Build Commands

```bash
make serve          # Build frontend + serve-rs тЖТ http://localhost:8080
make serve-dev      # Dev mode (unoptimized WASM + debug server)
make web            # Build Leptos/WASM only тЖТ web/dist/
make web-dev        # Build Leptos/WASM unoptimized
make release        # Package .app + .dmg for macOS
make release-macos  # Same as release
make clean          # Remove build/, web/dist/, dist/
```

## Architecture Overview

Plugin-first local LLM chat interface. The core provides a minimal chat + LLM config UI; everything else is plugins.

### Stack

- **Backend**: Rust (`server/`) тАФ Axum HTTP server (`serve-rs`) serving the WASM frontend and exposing REST endpoints
- **Frontend**: Rust/WASM (`web-app/`) тАФ Leptos CSR app compiled to WASM via Trunk
- **Runtime**: `wry` + `tao` тАФ native WebView window wrapping `localhost:PORT`
- **State**: persisted in `~/.local-ai/` (settings, plugins, uploads, workspace)

### Project Structure

```
server/       Rust HTTP server (serve-rs)
web-app/      Leptos WASM frontend
web/dist/     Built frontend output (generated by trunk)
scripts/      Release packaging (package-macos.sh)
plugins/      Plugin packages (e.g. oliv4600-pack)
dist/         Release artifacts (.app, .dmg)
```

### Server (`server/src/`)

| File | Role |
|------|------|
| `main.rs` | Entry point; spins HTTP server in background thread, opens wry WebView |
| `config.rs` | Config from CLI flags тЖТ env vars тЖТ defaults. `ai_base = ~/.local-ai/` |
| `routes/chat.rs` | LLM streaming proxy (SSE) |
| `routes/plugins.rs` | Serves installed plugins from `~/.local-ai/plugins/` |
| `routes/settings.rs` | LLM endpoint/model config persisted to `~/.local-ai/settings.json` |
| `routes/static_files.rs` | Serves `web/dist/` as SPA fallback |

### Frontend (`web-app/src/`)

| File | Role |
|------|------|
| `lib.rs` | WASM entry point, mounts `App` |
| `shell.rs` | Main layout: sidebar (only if plugins installed) + content area |
| `chat.rs` | Chat panel |
| `config_panel.rs` | LLM settings panel |
| `plugins/mod.rs` | `PluginInfo` type for plugin descriptors |

### Default behavior (no plugins)

- Full-screen chat with floating settings button
- Settings panel: configure LLM endpoint + model
- No sidebar shown

### With plugins installed

- Sidebar appears with icon buttons: Chat, [plugins...], Settings
- Plugin content rendered in `<iframe src="/plugins/{id}/app/">`
- Plugins live in `~/.local-ai/plugins/{id}/` with a `plugin.json` manifest

### Plugin system

Plugins are directories installed to `~/.local-ai/plugins/`. Each plugin has:
- `plugin.json` тАФ manifest (`id`, `name`, `icon`)
- `app/` тАФ static web app served by the server at `/plugins/{id}/app/`

### LLM connection

Connects to any OpenAI-compatible API. Default: `http://localhost:11434` (Ollama). Configure in the app's settings panel or via `LLM_ENDPOINT` env var.

### Release (macOS)

`make release-macos` тЖТ `scripts/package-macos.sh`:
1. `cargo build --release` тЖТ `serve-rs` binary
2. `cargo clean --target wasm32-unknown-unknown` + `trunk build --release` тЖТ `web/dist/`
3. Assembles `LocalAiAssistant.app` bundle (serve-rs + web/dist + launcher script)
4. Creates `.dmg` via `hdiutil`

On launch, the app creates `~/.local-ai/{plugins,workspace,uploads}/` if they don't exist.
